# 버전명 정책

이 저장소의 릴리즈 버전은 사용자가 받은 배포 파일을 쉽게 식별하고, 문제가 생겼을 때 같은 배포본을 다시 찾을 수 있도록 정합니다.

## 기본 형식

베타 배포는 다음 형식을 사용합니다.

```text
beta-YYYYMMDD-vN
```

예시:

```text
beta-20260519-v1
beta-20260519-v2
beta-20260520-v1
```

- `YYYYMMDD`: 배포일입니다.
- `vN`: 같은 날짜에 다시 배포할 때 1부터 증가시킵니다.
- Git tag, GitHub Release 제목, 릴리즈 zip 설명에는 같은 버전명을 사용합니다.

## 언제 번호를 올리는가

같은 날짜에 아래 항목 중 하나라도 바뀌면 `vN`을 올립니다.

- 배포 zip 내용
- SHA-256 체크섬
- 런처 동작
- 번역/패치 데이터
- 사용자에게 영향을 주는 안내 문서

문서 오타처럼 사용자 동작이나 배포 zip 식별에 영향이 없는 수정은 같은 릴리즈를 다시 만들지 않고 다음 배포에 포함합니다.

## 이전 임시 표기

기존 임시 배포에서 사용한 `v2026.05.07` 같은 날짜형 표기는 과거 배포 식별용으로만 유지합니다.

새 배포부터는 베타 상태가 명확히 드러나는 `beta-YYYYMMDD-vN` 형식을 우선 사용합니다.

## 정식 릴리즈 전환

베타가 끝나고 안정 배포로 전환할 때는 별도 기준을 정한 뒤 `v1.0.0` 같은 semantic version 형식으로 전환할 수 있습니다. 그 전까지는 날짜 기반 베타 버전을 사용합니다.
