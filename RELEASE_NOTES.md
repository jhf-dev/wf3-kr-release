# 릴리즈 노트 (beta-20260520-v1)

## 다운로드

- 배포 파일: `wind3-korean-patch-beta-20260520-v1.zip`
- SHA-256: `0EB99653F6B99FC269DC40931021D492A350F450261FF40D6A262F99767C8A03`
- 파일 크기: `18,953,478` bytes

## 주요 변경사항

* 창모드 멀티태스킹 개선
  - 전체 창모드와 창 모드에서 Alt+Tab 이후 작업표시줄 뒤로 밀리는 문제를 줄이고, 활성 상태에서만 topmost가 유지되도록 조정했습니다.
  - 게임 창이 비활성일 때는 마우스 중심점 고정, ClipCursor, SetCursor(NULL) 계열 점유 동작을 완화해 Windows 커서 전환이 자연스럽게 돌아오도록 했습니다.
  - 포커스 복귀 직후 첫 마우스 이동에서 큰 delta가 들어가지 않도록 중심점 재동기화 보정을 추가했습니다.

* 비활성 상태 화면 갱신 옵션
  - `runtime_policy.ini`의 `inactive_pause_enabled=0` 설정으로 게임 창이 비활성일 때도 렌더 루프가 계속 돌도록 하는 런타임 보정을 추가했습니다.
  - 이 경로는 Ghidra 기반 정적 분석 결과와 런타임 로그 게이트로 확인할 수 있게 했습니다.

* 해상도 설정 정리
  - 런처에서 전체 창모드, 창 모드, 독점 전체화면을 선택하고 해상도를 저장할 수 있게 했습니다.
  - 전체 창모드는 4:3 기준의 integer 스케일과 fit 스케일을 선택할 수 있습니다.
  - 창 모드는 현재 모니터 작업 영역 안에 들어가는 주요 4:3 해상도 프리셋을 제공합니다.
  - WindConfig 호환 레지스트리 값(`IsFullscreen`, `CreationWidth`, `CreationHeight`)을 런처 실행 전에 반영합니다.

* 배포 런처 UI 정리
  - 배포용 런처에서는 개발용 런타임 수동 제어를 숨기고, 게임 실행, 해상도 설정, 진단 정보 저장 중심으로 정리했습니다.
  - 개발용 런처에는 기존 고급 런타임 설정을 남겨 실험과 회귀 확인이 가능하도록 했습니다.

* 로드 직전 멈춤 완화
  - 기본 safe 런타임에서는 source-text 복구용 `prepare-load`를 실행하지 않도록 조정했습니다.
  - 로드 직전 오래 대기하던 경로를 줄이고, 로드 후 필요한 UI 라벨 재적용만 다시 예약합니다.

* 런타임 라벨 적용 비용 축소
  - 캐릭터 정보창과 스킬 화면 라벨을 적용한 뒤 같은 영역을 다시 dry-run 검증하던 중복 스캔을 릴리즈 기본 경로에서 제거했습니다.
  - 라벨 적용 보고와 패키지 검증은 유지하면서, 실제 실행 중 로드 후 처리량을 줄이는 쪽으로 정리했습니다.

* 배포 환경 런타임 helper 경로 수정
  - 깨끗한 설치 환경에서 런타임 패치 helper가 개발용 `analysis_out` 폴더를 찾다가 실패하던 문제를 수정했습니다.
  - 이제 패키지에 포함된 `_runtime/worker/analysis_out`의 helper 실행 파일을 사용하므로, 개발 PC에만 있던 도구 파일에 의존하지 않습니다.

* 스킬/정보창 잔여 라벨 적용 경로 보강
  - 스킬 화면의 `잔여점수`, 캐릭터 정보창의 `설명`, `소유물품` 라벨이 기본 safe 런타임 패치 경로에 포함되도록 보강했습니다.
  - 릴리즈 ZIP 안에 필요한 helper 실행 파일과 런타임 정책 파일이 들어있는지도 빌드 단계에서 확인합니다.

* 캐릭터 정보창 좌측/중앙 스탯 라벨 재보강
  - 일부 PC에서 캐릭터 정보창의 `근력`, `체질`, `속도`, `기교`, `지력`, `매력`, `행운`, `신념` 라벨이 원문으로 남던 문제를 다시 보강했습니다.
  - 이전 live 검증에서 확인한 실제 적용 표면이 릴리즈 DLL, 런처 asset, 최종 ZIP까지 그대로 포함되도록 검증을 강화했습니다.

* 캐릭터 정보창 좌측/중앙 스탯 라벨 추가 보강
  - 이전 live 검증에서 성공했던 스탯 라벨 적용 경로가 릴리즈 DLL과 배포 ZIP까지 빠짐없이 포함되도록 보강했습니다.
  - `힘`, `체질`, `속도`, `기교`, `지력`, `매력`, `행운`, `신념` 라벨이 일부 환경에서 원문으로 남던 문제를 다시 정리했습니다.

* 스킬 화면 잔여 라벨 보강
  - 스킬 화면의 `잔여점수`, `설명:`처럼 별도 UI 캐시로 남던 라벨을 보강했습니다.
  - 넓은 범위의 재스캔이 아니라, 기존 실험 기록의 정확한 바이트 기준으로 필요한 라벨만 적용하도록 제한했습니다.

* 릴리즈 승격 검증 기준 강화
  - live에서 최종 성공을 확인한 표면은 다시 실험하지 않고, durable source, 빌드 DLL, 런처 asset, release ZIP에 같은 old/new byte와 site 경로가 들어갔는지 확인하도록 기준을 강화했습니다.
  - 다른 PC에서만 실패하는 경우 먼저 릴리즈 산출물 누락 여부를 확인하도록 지침을 추가했습니다.

* 캐릭터 정보창 스탯 라벨 추가 보강
  - 다른 PC에서 캐릭터 정보창 스탯 라벨이 원문으로 남던 문제를 추가 보강했습니다.
  - 스탯 라벨 문자열의 실제 게임 출력 형식에 맞춰 런타임 owner rewrite 매칭을 다시 조정했습니다.

* 릴리즈 전 산출물 검증 강화
  - 런타임에서 확인된 수정이 최종 배포 ZIP까지 들어갔는지 별도로 검증하도록 기준을 강화했습니다.
  - 패키지 안에 필요한 런타임 적용 경로와 DLL 내부 표식이 빠지면 릴리즈 빌드가 실패하도록 보강했습니다.

* 캐릭터 정보창 기본/장비 라벨 추가 보강
  - 다른 PC에서 캐릭터 정보창의 `레벨`, `직업레벨`, `생명`, `정신`, `투지`, 장비 슬롯 라벨 일부가 원문으로 남던 문제를 보강했습니다.
  - 런타임 owner rewrite가 실제 게임 문자열의 콜론 포함 형식과 맞도록 수정했습니다.

* 진단 로그 수집 범위 확대
  - `진단 정보 저장` 결과에 startup bridge 내부 로그, 런타임 패치 cycle report, worker stdout/stderr 로그를 추가했습니다.
  - 다른 PC에서만 실패하는 경우에도 어느 적용 단계에서 실패했는지 더 빨리 확인할 수 있습니다.

* 다른 PC에서의 런타임 DLL 로드 안정성 보강
  - 패치용 `d3d9.dll`의 외부 C 런타임 의존성을 제거해, PC 환경 차이로 DLL이 로드되지 않을 가능성을 줄였습니다.
  - 기존 기능인 캐릭터 정보창 스탯 표기 복구와 런타임 기능 설정은 유지됩니다.

* 원격 환경 진단 기능 추가
  - 런처에 `진단 정보 저장` 기능을 추가했습니다.
  - 다른 PC에서만 문제가 발생할 때 선택한 게임 폴더, 주요 파일 해시, 런타임 설정, 관련 로그, 최근 Windows 오류 기록을 한 번에 저장할 수 있습니다.

* 캐릭터 정보창 스탯 표기 복구
  - 일부 환경에서 캐릭터 정보창의 스탯 라벨이 원문으로 되돌아가던 문제를 수정했습니다.
  - 엔진 레벨의 owner trace/rewrite 적용 경로가 릴리즈 DLL에 포함되도록 바로잡았습니다.

* 런타임 기능 설정 방식 개선
  - `runtime_policy.ini`의 `[runtime_features]` 섹션에서 owner trace/rewrite, 모듈 UI 패치, 진단용 프로브 같은 런타임 기능을 제어할 수 있게 했습니다.
  - 같은 종류의 기능 전환을 위해 매번 DLL을 새로 빌드해야 하는 부담을 줄였습니다.

* 릴리즈 패키징 검증 강화
  - 릴리즈 빌더가 Phase 1C owner-rewrite용 런타임 DLL인지 확인한 뒤 패키징하도록 검증을 추가했습니다.
  - 잘못된 DLL이 다시 배포 ZIP에 섞이는 상황을 조기에 차단합니다.

* 런타임 영역 패치 적용 방식 조정
  - 런처 실행 직후와 로드 이후 UI 패치가 더 빠르게 반영되도록 적용 타이밍을 보강했습니다.

* 스토리 번역 품질 1차 폴리싱
  - 주요 대사의 표현, 말투, 용어를 1차로 정리해 읽기 흐름을 개선했습니다.

* 일부 미번역 대응
  - 플레이 중 노출되던 UI, 이름, 항목 텍스트 일부에 한국어 대응을 추가했습니다.

* 챕터 제목 이미지 한글화
  - 이미지로 표시되는 챕터 제목 일부를 한국어로 교체했습니다.

* 런처 게임 경로 감지 기능 추가
  - Steam 설치 경로에 있는 `Wind Fantasy 3` 폴더를 자동으로 감지하는 기능을 추가했습니다. 폴더명이 다른 경우 수동 선택이 필요할 수 있습니다.

## 사용 전 주의사항

- 이 배포 파일은 게임 본편이나 원본 게임 리소스를 포함하지 않습니다. 사용자가 보유한 Wind Fantasy 3 설치본이 필요합니다.
- 스팀 클라이언트 기준 2025-08-12 버전을 기반으로 개발했습니다. 이외 버전에서는 패치 적용 및 동작을 보장하지 않습니다.
- 패치 적용 전에는 게임을 완전히 종료해 주세요. 게임이 실행 중이면 일부 파일 교체가 실패하거나 적용 결과가 일관되지 않을 수 있습니다.
- 중요한 저장 데이터는 패치 적용 전에 별도로 백업하는 것을 권장합니다.
- 이 패치는 비공식 개인 한국어 패치입니다. 모든 PC 환경에서 동일하게 동작한다고 보장하지 않습니다.
- 일부 이미지형/헤더형 UI 문구는 아직 후속 보강 대상일 수 있습니다.
